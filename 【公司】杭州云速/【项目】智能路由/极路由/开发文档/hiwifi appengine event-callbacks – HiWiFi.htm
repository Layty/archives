路由器事件回调方法 ¶

App Engine 提供了将系统事件回调方法（callback）的注册、删除方法。

注册脚本到某类型的系统事件 ¶

  • 方法一： 
    sysevent 事件类型 回调脚本文件
  • 方法二： 
    echo "脚本内容" | sysevent 事件类型 -
  • 参数说明：
    • 事件类型：有效值为 iface、block、button、usb；
    • 回调脚本：传入已准备好的回调脚本的文件，可为临时文件；文件名可传入“-”，从标准输入读入

删除事件回调 ¶

sysevent -r

其他说明 ¶

  • 传入的环境变量及举例 
    ACTION=ifup       # 事件行为；“iface”的有效值为 ifup、ifdown；“block”的有效值为 add、remove；
DEVICE=eth1       # [iface]网络接口的设备名称，例如：eth1、wlan0
INTERFACE=wan     # [iface]网络接口的OpenWrt名称，例如：wan、lan、lan2
DEVPATH=/dev/sda1 # [block]可插拔存储的设备文件名
  • 举例
    • 例如需截获每个网口每次up的事件，并将网口IP地址已文件名“ip”存放到当前用户home目录下，则可定义脚本： 
      #!/bin/sh
if [ "$ACTION" = "ifup" ]; then
    . /lib/functions/hiwifi-network.sh
    ip=`get_iface_ipaddr "$DEVICE"`
    echo "$ip" > ~/ip
fi
      假设命名为getip.sh，则执行如下命令进行注册： 
      sysevent iface getip.sh
    • 例如需截获每个USB存储分区插入、移除的事件，则可定义脚本： 
      #!/bin/sh

device=`basename $DEVPATH`
case "$device" in
  sd*)
    case "$ACTION" in
      add) ......;;
      remove) ......;;
    esac
    ;;
esac
      假设命名为storage-detect.sh，执行： 
      sysevent block storage-detect.sh
