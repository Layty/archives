HiWiFi App Engine - App开发方法 ¶

App基本开发相关说明 ¶

  • 一个典型的App包含至少一个安装、卸载、控制脚本：script。
    插件安装成功后，该脚本会被App Engine拷贝为 ~/config/script （~表示App的home目录）。
    该脚本的构成如下。 
    #!/bin/sh

start()
{
    # App启动代码
    # 函数返回0表示启动成功，返回非0表示启动失败
}

stop()
{
    # App停止代码
}

status()
{
    # App状态查询代码
    if <运行中>; then
        echo "{ \"status\" : \"running\" }"
    else
        echo "{ \"status\" : \"stopped\" }"
    fi
}

install()
{
    # App安装代码
    # 函数返回0表示安装成功，返回非0表示安装失败
}

uninstall()
{
    # App卸载代码
}
  • App安装时需获得通过云平台配置参数的方法

假定名为“dnspod”的App在云平台设定了 DNSPODACCOUNT, DNSPODPASSWORD, DNSPODDOMAIN, DNSPODSUBDOMAIN, DNSPODIPFROM 5个参数，则增加文件“dnspod.conf”作为参数接收模板，内容为： 

DNSPODACCOUNT={$DNSPODACCOUNT}
DNSPODPASSWORD={$DNSPODPASSWORD}
DNSPODDOMAIN={$DNSPODDOMAIN}
DNSPODSUBDOMAIN={$DNSPODSUBDOMAIN}
DNSPODIPFROM={$DNSPODIPFROM}

另假设这5个参数被分别设定为 hello@…, hello123, hiwifi.com, www, 1，则在插件安装时该文件会内容会被替换为： 

DNSPODACCOUNT='hello@abc.com'
DNSPODPASSWORD='hello123'
DNSPODDOMAIN='hiwifi.com'
DNSPODSUBDOMAIN=www''
DNSPODIPFROM='1'

在 script 脚本引用参数之前，通过以下代码将该配置文件包含进来： 

. $HOME/config/dnspod.conf

【特别提示】 .conf 主文件名必须与插件名称一致，否则变量不会被替换。

  • App安装脚本中可通过环境变量获取的参数 
    DEVICE_RID   # 设备在云平台的ID
SID          # 当前安装App的唯一ID
VERSION      # 当前安装App的版本

插件安装时的执行方式 ¶

  • App安装时，script脚本以如下方式调用 
    . /etc/environment
. ./script
install
  • App安装引擎会捕获“install”方法的返回值，返回非0表示安装失败。
  • 由于安装脚本是以include（包含）方式引用并执行的，故script脚本除了定义install/uninstall等函数之外无须添加任何入口代码，例如以下用法是【错误】的： 
    install() {
  ...
  ...
}
uninstall() {
  ...
  ...
}

install

App上线前的调试方法 ¶

将插件相关的文件上传至路由器一空目录下，cd 到该目录，执行： 

( . ./script; install )

若App的install方法内未自动启动（即未调用start），则需执行以下代码启动： 

( . ./script; start )

AppEngine中关联系统配置文件的方法 ¶

部分应用需要修改系统基础服务的配置，但这些服务的配置文件位于主文件系统中，并大多在主文件系统中以root权限运行，而AppEngine以普通用户权限运行于chroot隔离的文件系统中，在AppEngine中不能直接存取这些配置文件也不能直接重启这些服务。因此AppEngine为这些服务定义了配置文件关联方法。

  1. 目前支持的配置文件类型：
    • ~/etc/dnsmasq.d：dnsmasq规则，主要用于：
      • 将某部分域名使用指定的DNS服务器进行解析；
      • 将某部分域名解析到的IP添加到指定的salist地址表（暂未开放）；
    • ~/etc/kproxy.d：kproxy规则，极路由独有layer-7代理服务，判断将何种流量定向到nginx；
    • ~/etc/iptables.d：自定义iptables的shell脚本，会在系统firewall加载完毕后遍历执行（以hiwifi帐号权限执行）；
    • ~/etc/hotplug.d/network：网络接口up/down事件的回调脚本，具体编写方法请参见OpenWrt相关资料（以hiwifi帐号权限执行）；
  2. 重新加载服务配置的方法： 
    haecli config-reload [kproxy|dnsmasq|hosts|filewall|...]   # 重新加载当前应用所属的系统服务配置文件

其他说明 ¶

  • USB存储挂载位置: /tmp/storage。例如一个移动硬盘有3个分区，可在该目录下看到： 
    hiwifi@Hiwifi(AppEngine):~$ ls /tmp/storage/ -l
drwxrwxrwt    2 root     root            40 May 20 16:01 sda1
drwxrwxrwt    2 root     root            40 May 20 16:01 sda2
drwxrwxrwt    2 root     root            40 May 20 16:01 sda3
  • HiWiFi系统保留端口 
    TCP:
22       -- dropbear服务端口（SSH）
80       -- nginx管理页面服务
81,83    -- nginx HTTP代理端口（通常由kproxy匹配的HTTP请求重定向过来）
139,445  -- samba server
515      -- syslog
1022     -- AppEngine的dropbear服务端口
1723     -- PPTP client
5000     -- miniupnpd

UDP:
53       -- dnsmasq
137,138  -- samba server
515      -- syslog
1900     -- miniupnpd
  • 以下程序有setuid属性，执行时拥有root权限 
    ifconfig route ip iptables ip6tables
  • App安装失败时显示错误原因的方法 标准输出中，以“<User-Echo>”开头的行会被系统截获并在插件安装失败（install函数返回非0）时，显示在云平台。示例如下。 
    # Echo to web page
user_echo()
{
  echo "<User-Echo>$@"
}

install()
{
  ...
  ...
  if ! ping6 ipv6.baidu.com -c1 -w3; then
    echo "<User-Echo>错误原因：您无IPv6网络连接。"
    exit 1
  fi

}
